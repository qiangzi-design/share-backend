# 项目总结（2026-03-29 ~ 2026-03-31）

## 1. 复盘背景
本次迭代围绕“稳定性优先 + 可持续迭代”推进，目标是把项目从“可跑”提升到“可维护、可扩展、可持续上线”。

核心原则：
- 先打稳底座（接口规范、安全、异常、数据库一致性、测试基线）
- 再补业务（资料、搜索、内容管理、社交、私聊、通知）
- 兼顾用户体验（视觉、交互、反馈一致性）

## 2. 这几天完成的主要工作

## 3. 关键问题与解决方案

### 3.1 启动失败：Spring Bean 循环依赖
问题：`jwtAuthenticationFilter -> userServiceImpl -> securityConfig -> jwtAuthenticationFilter`。
解决：
- 把 `PasswordEncoder` 独立到 `PasswordConfig`。
- 梳理安全配置与用户服务依赖边界，解除循环。
结果：后端可正常启动。

### 3.2 SQL 脚本在 MySQL 报语法错误
问题：`ALTER TABLE ... ADD COLUMN IF NOT EXISTS` 在目标 MySQL 版本不兼容。
解决：
- 改为 `information_schema + PREPARE/EXECUTE` 的幂等迁移写法。
结果：迁移脚本可重复执行，兼容存量库。

### 3.3 通知表结构漂移导致插入失败
问题：历史库中列名/类型不一致（如 `user_id`、`type` 为整型），导致：
- `Incorrect integer value: 'content_like' for column 'type'`
- `Field 'user_id' doesn't have a default value`
解决：
- 在 Phase9 迁移中加入兼容迁移：列重命名、类型修正、缺失列补齐、索引幂等创建。
结果：通知写入恢复正常。

### 3.4 前端依赖错误：`echarts` 无法解析
问题：Vite 报 `Failed to resolve import "echarts"`。
解决：安装并锁定依赖，统一在 `package.json` 维护。
结果：图表页面可正常构建运行。

### 3.5 私聊角标更新慢、不同页面不同步
问题：角标存在轮询延迟，且消息页/私聊页状态不一致。
解决：
- WebSocket 实时事件直接驱动未读状态。
- 补充 `chat-unread-sync` / `interaction-unread-sync` 事件总线同步。
- 轮询仅作为兜底并缩短周期。
结果：角标更新更及时、跨页面一致。

### 3.6 登录状态跨浏览器串联
问题：新开浏览器进程仍显示已登录。
根因：登录态保存在 `localStorage`（跨标签页持久）。
解决：改为 `sessionStorage`。
结果：同一浏览器会话有状态，不同浏览器进程隔离。

### 3.7 视觉与可用性问题
问题：按钮不可见、边框被“吞”、样式不统一。
解决：
- 统一按钮样式变量与状态样式。
- 细化 hover/focus/active 边框与阴影。
- 统一主次按钮视觉层级。
结果：可读性与可点击性显著提升。

### 3.8 编码问题（乱码）
问题：部分文件出现中文乱码（错误转码污染）。
解决：
- 定位并修复关键页面文本。
- 做过一轮全量排查并清理主要乱码点。
现状：核心用户页面已恢复；历史文档仍有少量乱码，建议后续统一做编码治理。
